Quick Sort


Recursively selects pivot point, partitions array based on that pivot point, and selects new pivot in each of the partitioned arrays
Better for smaller arrays, and is in place (does not require extra memory)
Not practical for Linked Lists, can have bad worst case scenario on large data sets
Unstable
Good locality of reference (because all memory stored in same place, faster to access)
Time Complexity: Best/Average: O(N log N), Worst: O(N^2), Space Complexity: O(N log N)
